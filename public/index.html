How to Use GetApproved

1. First Time Setup

When you first use the /getapproved command, you'll be prompted to select a channel for logging approval activities. This helps maintain a clear audit trail.

2. Create an Approval Request

Type /getapproved in any channel or DM. Fill in the form with:

  • Select approvers from your team
  • Add the URL that needs review
  • Provide details about what needs approval

3. Review and Respond

Approvers will receive a direct message with the request and can:

  • Click Approve to accept
  • Click Reject to decline
  • View the status of other approvers

Privacy & Security

Data Collection & Usage

GetApproved collects and stores only the information necessary for the approval process:

  • Team and user IDs for routing messages
  • Selected logging channel for documentation
  • Approval request details (URL, description, approvers)
  • Approval status and decisions

We do not:

  • Store message contents beyond approval requests
  • Track user activity outside of approval actions
  • Share data with third parties
  • Use data for advertising or marketing
